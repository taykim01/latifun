무엇을 만드는가
-> 기획으로 프로덕트가 만들어지는 AI WB
    -> 기획은 무엇인지: WF, DB 스키마, UF, 서비스 기획서
    -> 어떤 프로덕트 스택을 사용할지: Next.js, Supabase, DDD
    -> 유저 인풋 -> 코드 생성 -> 코드 테스트/수정 -> 배포 -> 업데이트

왜 만드는가
-> 개발 리소스를 줄이고 프로덕트를 쉽고 빠르게 만들기 위함

누구를 위해 만드는가
-> 기획자



기능 정리
- signInUseCase: 로그인 기능
- signOutUseCase: 로그아웃 기능
- createProfileUseCase; 유저 프로필 생성하는 기능
- createProjectUseCase: 프로젝트 생성하는 기능
- generatePresentationStructureUseCase: /(pages) 구조 생성하는 기능
- generatePageUseCase: 각 페이지 생성하는 기능(레이아웃, 컨트롤러 포함)
- generateApplicationUseCase: Use Case 생성하는 기능
- testApplicationUseCase: Use Case 테스트하는 기능
- customizeComponentsUseCase: Shadcn UI 컴포넌트 커스터마이즈하는 기능
- generateSchemaUseCase: DB 스키마 생성하는 기능 + Type 업데이트 해줘야 함
